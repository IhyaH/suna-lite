# 文档结构总览

Suna-Lite 项目的文档按照功能和使用场景进行了分类整理，便于用户和开发者快速找到所需信息。

## 📁 文档目录结构

```
docs/
├── README.md                    # 主文档（项目完整说明）
├── guides/                      # 用户指南
│   ├── INSTALLATION.md          # 安装指南
│   └── CONFIG_GUIDE.md          # 配置指南
├── reference/                   # 参考资料
│   ├── CLAUDE.md                # 开发者指南
│   └── PROJECT_STRUCTURE.md     # 项目结构说明
├── development/                 # 开发文档（待补充）
├── INSTALLATION.md              # 安装指南（保留在根目录以便访问）
├── USAGE.md                     # 使用说明
├── CONFIGURATION.md             # 配置说明
├── EXAMPLES.md                  # 使用示例
└── ARCHITECTURE.md              # 架构文档
```

## 📖 文档分类说明

### 🚀 用户指南 (guides/)
这些文档为最终用户提供使用指导：
- **安装指南** - 详细的安装步骤和环境配置
- **配置指南** - 系统配置和参数说明

### 📚 参考资料 (reference/)
这些文档为开发者提供技术参考：
- **开发者指南** - 开发规范和最佳实践
- **项目结构** - 完整的代码结构说明

### 🛠️ 开发文档 (development/)
开发相关的技术文档（待补充）

### 📋 核心文档
直接位于docs/目录下的核心文档：
- **使用说明** - 完整的功能使用指南
- **配置说明** - 详细的配置选项说明
- **使用示例** - 实际使用案例和最佳实践
- **架构文档** - 系统架构设计说明

## 🔍 快速查找

| 首次使用 | [安装指南](guides/INSTALLATION.md) → [使用说明](USAGE.md) |
| 配置系统 | [配置指南](guides/CONFIG_GUIDE.md) → [配置说明](CONFIGURATION.md) |

| 问题排查 | [使用示例](EXAMPLES.md) → [使用说明](USAGE.md) |

## 📝 文档维护

- 所有文档文件都使用Markdown格式
- 文档应该保持更新，与代码同步
- 新功能添加时应该及时更新相关文档
- 文档应该包含清晰的示例和说明

## 🤝 贡献文档

如果您想为项目贡献文档，请：

1. 阅读开发者指南了解文档规范
2. 在相应的目录下创建或更新文档
3. 确保文档格式统一，内容准确
4. 提交Pull Request进行审核
